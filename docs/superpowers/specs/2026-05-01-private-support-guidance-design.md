# 私域联系方式引导设计方案

## 背景与目标

当前产品处于小范围邀测阶段，用户在生成图片时可能需要额度咨询、失败处理、提示词优化和充值沟通。为了把高意向用户引导到可运营的私域，需要在创作台提供 QQ 和微信联系方式。

本次目标：

- 在用户主要使用路径上展示私域联系方式。
- 不阻断图像生成流程。
- 联系方式可通过环境变量配置，避免写死在代码里。
- 没有配置联系方式时页面仍可正常渲染。

## 方案

在创作台生成表单底部增加“专属服务”面板，展示 QQ 和微信两个联系入口。面板文案聚焦用户收益：额度咨询、生成失败处理、提示词优化。

联系方式从前端构建环境变量读取：

- `VITE_PRIVATE_SUPPORT_QQ`
- `VITE_PRIVATE_SUPPORT_WECHAT`

如果某个值未配置，显示“待配置”。这样本地开发、测试和生产部署都不会因为缺少私域配置而失败。

## 非目标

- 不做扫码图片上传或二维码管理。
- 不做复制按钮、跳转微信或唤起 QQ 客户端。
- 不做后台运营配置页面。
- 不拦截未加私域用户的生图流程。

## 验收标准

- 创作台展示“专属服务”区域。
- 区域中包含 QQ 和微信联系方式。
- 联系方式可通过 Vite 环境变量配置。
- 未配置联系方式时显示占位文案。
- 前端测试和生产构建通过。
